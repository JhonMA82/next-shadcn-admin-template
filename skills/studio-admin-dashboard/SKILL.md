## When to Use

- Creating a new dashboard page (CRM, Analytics, Finance, etc.)
- Adding items to the sidebar navigation
- Building dashboard-local components (`_components/`)
- Adding TanStack Table data tables to a dashboard
- Creating a placeholder "coming soon" dashboard stub

## Critical Patterns

### Dashboard page = 2 files touched, 1 folder created

| Action | File to modify |
|--------|---------------|
| Add sidebar menu entry | `src/navigation/sidebar/sidebar-items.ts` |
| Create the page | `src/app/(main)/dashboard/<name>/page.tsx` |
| Add local components | `src/app/(main)/dashboard/<name>/_components/*.tsx` |

**Nothing else.** Layout, theme, search, and preferences are all inherited automatically.

### Server Component by default

`page.tsx` is always a Server Component — NO `"use client"` directive. Add it only to child components that need browser APIs.

```tsx
// CORRECT — page.tsx
import { MyCard } from "./_components/my-card";

export default function Page() {
  return (
    <div className="@container/main flex flex-col gap-4 md:gap-6">
      <MyCard />
    </div>
  );
}
```

### "use no memo" for TanStack Table

Tables using `@tanstack/react-table` need BOTH directives:

```tsx
"use client";
"use no memo";

import { useReactTable } from "@tanstack/react-table";
```

React Compiler memoization conflicts with TanStack's internal hooks. Without `"use no memo"`, table state (sorting, filtering) breaks silently.

### Named exports for all components

Never use `export default` for components. The only default export in the app is `Page()` in `page.tsx`.

```tsx
// CORRECT
export function MyCard() { ... }

// WRONG
export default function MyCard() { ... }
```

### Sidebar navigation: single source of truth

`src/navigation/sidebar/sidebar-items.ts` is the ONLY file that defines menu structure. Search dialog auto-indexes from it. No other file needs changes.

```typescript
// Add to the appropriate group in sidebarItems array
{
  title: "My Dashboard",
  url: "/dashboard/my-dashboard",
  icon: Store,  // from lucide-react, uninstantiated
},
```

### "Coming Soon" stub pattern

For placeholders, use `comingSoon: true` and point to the shared stub:

```typescript
{
  title: "Future Feature",
  url: "/dashboard/coming-soon",
  icon: Rocket,
  comingSoon: true,
},
```

The stub page at `src/app/(main)/dashboard/coming-soon/page.tsx` is already built. Search dialog automatically filters out `coming-soon` URLs.

### Do NOT edit these directories

| Path | Reason |
|------|--------|
| `src/components/ui/` | shadcn auto-generated, excluded from Biome |
| `src/lib/preferences/theme.ts` (generated section) | Auto-regenerated by `pnpm generate:presets` |

## Component File Structure

### Simple dashboard (static cards, charts)

```
src/app/(main)/dashboard/<name>/
├── page.tsx                    # Server Component
└── _components/
    ├── metric-cards.tsx        # Static display component
    ├── chart-overview.tsx      # "use client" for recharts
    └── data.json               # Mock data (optional)
```

### Dashboard with data table

```
src/app/(main)/dashboard/<name>/
├── page.tsx
└── _components/
    └── <table-name>/
        ├── schema.ts           # Zod schema + type export
        ├── columns.tsx         # "use client"; "use no memo";
        ├── table.tsx           # "use client"; "use no memo";
        └── data.json           # Mock data
```

`schema.ts`:
```typescript
import z from "zod";

export const rowSchema = z.object({
  id: z.string(),
  name: z.string(),
  status: z.string(),
});

export type Row = z.infer<typeof rowSchema>;
export const rowsSchema = z.array(rowSchema);
```

`columns.tsx`:
```tsx
"use client";
"use no memo";

import type { ColumnDef } from "@tanstack/react-table";
import type { Row } from "./schema";

export const columns: ColumnDef<Row>[] = [
  { accessorKey: "id", header: "ID" },
  { accessorKey: "name", header: "Name" },
  { accessorKey: "status", header: "Status" },
];
```

## Data Patterns

### Option A: Inline constants (simple dashboards)

```typescript
const ROWS = [
  { id: 1, label: "Revenue", value: "$12,430" },
  { id: 2, label: "Users", value: "1,240" },
] as const;
```

### Option B: JSON imports (medium complexity)

```typescript
import data from "./data.json";
// Optional Zod validation:
import { rowsSchema } from "./table/schema";
const rows = rowsSchema.parse(data);
```

## Biome Formatting Rules

- Double quotes in JSX: `className="flex"`
- Semicolons: always
- Trailing commas: always
- Self-closing elements: `<Input />`
- Sorted Tailwind classes
- No `any` type
- Named exports, not default (except page.tsx)

## Commands

```bash
pnpm dev              # Start dev server
pnpm check            # Run Biome CI check (lint + format, no write)
pnpm check:fix        # Auto-fix Biome issues
pnpm generate:presets # Regenerate theme presets (run after adding CSS preset)
```

## Quick Decision Table

| I need to... | Touch these files |
|-------------|-------------------|
| Add a new dashboard | `sidebar-items.ts` + `page.tsx` |
| Add a table to a dashboard | `_components/<name>/{schema,columns,table}.tsx` |
| Add mock data | `_components/data.json` |
| Add a "coming soon" placeholder | `sidebar-items.ts` only (`comingSoon: true`) |
| Add a shared UI component | `src/components/<name>.tsx` |
| Change app name | `src/config/app-config.ts` |
| Add a theme preset | `src/styles/presets/<name>.css` → `pnpm generate:presets` |
| Change default preferences | `src/lib/preferences/preferences-config.ts` |
